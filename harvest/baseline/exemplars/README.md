# Synthetic Reference Set — Exemplars

The neutral, task-matched yardstick each battery brief's target output is judged
against. One Fable-authored exemplar per brief, **broad-craft-sourced** from the
human writing in `~/makegood-harvest-sources/` (real, general, cross-domain — not
nonprofit) and generated with **no S0 / F0 / backstop / tic list** in the
generator context. The exemplar is a comparison instrument, never the authority
on what is human, and it is synthetic-to-test, not a copied human sample.

## State (2026-07-17): being rebuilt

The original 12 exemplars (F01–F12) were **deleted** with the battery they matched
— that battery was scoped to a single nonprofit org in one promotional register
and could only surface tics within that genre. See
[`../../battery/COVERAGE_SPEC.md`](../../battery/COVERAGE_SPEC.md) for the
replacement battery design (domain/register coverage, no fixed substrate,
stable-core + growing-pool) and `[[harvest-battery-purpose]]` for why.

**Nothing here is frozen or approved.** New exemplars are regenerated (one per new
brief, same method — Fable, broad-craft-sourced, guardrail-free) once the new
briefs are written and approved. This README gets its provenance table and layout
back at that point.

## The mechanism (carries forward unchanged)

Target and exemplar write the same brief, so a difference between them is
attributable to *how it's written*, not the topic. The exemplar generator is a
different model than the targets under test — the cross-generation guard that
keeps the reference from mirroring the model being measured.
